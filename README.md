# Многопоточная обработка данных с использованием ReentrantLock

Простое Java-приложение, демонстрирующее управление конкурентным доступом к общему ресурсу (массиву целых чисел) с помощью `ReentrantLock` и `ExecutorService`.

## Особенности
* **Потокобезопасность**: Использование `ReentrantLock` гарантирует, что только один поток может изменять массив данных в конкретный момент времени.
* **Пул потоков**: Эффективное управление рабочими потоками с помощью `ExecutorService`.
* **Разделение ресурсов**: Демонстрация взаимодействия между независимыми компонентами (Репозиторий, Обработчик и Главный класс).

## Структура проекта
Приложение состоит из трех основных классов внутри пакета `app`:
* **`Main.java`**: Точка входа в программу. Инициализирует ресурсы, создает фиксированный пул из 2 потоков, отправляет задачи на выполнение и управляет жизненным циклом экзекьютора.
* **`DataRepository.java`**: Выступает в роли источника данных, предоставляя исходный массив чисел `{7, 4, 5, 2}`.
* **`DataHandler.java`**: Содержит бизнес-логику. Безопасно блокирует общий массив, умножает каждый элемент на 3, выводит шаги трансформации в консоль и снимает блокировку.

## Принцип работы
1. Класс `Main` получает целевой массив из `DataRepository`.
2. Две конкурирующие задачи отправляются на выполнение в `ExecutorService`.
3. Первый поток, успевший захватить `ReentrantLock`, блокирует доступ для второго потока и начинает обработку.
4. Для каждого элемента массива поток сохраняет начальное значение, умножает его на 3, обновляет ячейку памяти и выводит результат.
5. Как только первый поток завершает работу и освобождает блокировку, второй поток захватывает ее и повторяет процесс умножения (уже над измененными числами).

## Ожидаемый вывод
При запуске приложения в консоли отобразится пошаговое изменение элементов массива:

```text
Initial value is 7
New value is 21
Initial value is 4
New value is 12
Initial value is 5
New value is 15
Initial value is 2
New value is 6
```
